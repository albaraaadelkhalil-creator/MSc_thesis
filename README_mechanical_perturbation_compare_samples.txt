README — mechanical_perturbation_compare_samples.py
===================================================

Purpose
-------

mechanical_perturbation_compare_samples.py compares selected mechanical
responses across multiple GUV photobleaching experiments.

This script is the second step of the workflow. It does not analyze TIFF images
or JFilament contours directly. Instead, it reads the CSV files produced by
ablation_analysis.py for each individual GUV.

Required preceding step
-----------------------

Before using this script, run ablation_analysis.py separately for every GUV or
experimental condition being compared.

Each individual analysis folder must contain:

    global_metrics_relative_to_prebleach.csv
    regional_metrics_relative_to_prebleach.csv

The intended workflow is:

    registered TIFF + JFilament contour
                    |
                    v
           ablation_analysis.py
                    |
                    v
       guv_deformation_analysis folder
                    |
                    v
    mechanical_perturbation_compare_samples.py
                    |
                    v
          cross-experiment comparison plots

Generated comparison plots
--------------------------

The script produces only the following four comparisons:

1. Global contour solidity.
2. Area and perimeter change relative to prebleach.
3. Mean absolute radial displacement in the bleached and non-bleached regions.
4. Mean absolute local strain in the bleached and non-bleached regions.

No velocity, Euclidean displacement, anisotropy, or additional deformation
plots are generated by this version.

Software requirements
---------------------

Use Python 3.10 or later. Required packages are:

    matplotlib
    numpy
    pandas

Install them with:

    pip install matplotlib numpy pandas

Input folder structure
----------------------

Each experiment entry should point to an output folder produced by
ablation_analysis.py.

Example:

    FRAP 4 lasers/
        guv_deformation_analysis/
            global_metrics_relative_to_prebleach.csv
            regional_metrics_relative_to_prebleach.csv

    FRAP 2 lasers 3 frames/
        guv_deformation_analysis/
            global_metrics_relative_to_prebleach.csv
            regional_metrics_relative_to_prebleach.csv

The script searches flexibly for the current filenames and also contains
fallback support for some older names, but the current filenames above are
preferred.

How to configure the script
---------------------------

Open mechanical_perturbation_compare_samples.py and edit the EXPERIMENTS list.

Each experiment requires:

    name
        Text used in plot legends.

    folder
        Path to the guv_deformation_analysis output folder created by
        ablation_analysis.py.

Example:

    EXPERIMENTS = [
        {
            "name": "4 lasers, 4 bleaching frames",
            "folder": Path(
                r"C:\path\to\experiment_1\guv_deformation_analysis"
            ),
        },
        {
            "name": "2 lasers, 3 bleaching frames",
            "folder": Path(
                r"C:\path\to\experiment_2\guv_deformation_analysis"
            ),
        },
    ]

You can add more experiments by adding more dictionaries to the list. Every
experiment must contain compatible CSV outputs from ablation_analysis.py.

OUTPUT_FOLDER
-------------

Set the folder in which the comparison figures will be stored:

    OUTPUT_FOLDER = Path(r"C:\path\to\comparison_output")

The folder is created automatically if it does not exist.

Length-unit setting
-------------------

PREFERRED_LENGTH_UNIT determines which radial-displacement columns are used:

    PREFERRED_LENGTH_UNIT = "um"

or:

    PREFERRED_LENGTH_UNIT = "px"

Use "um" only when every individual ablation analysis used a valid
PIXEL_SIZE_UM value. Do not compare micrometre values from some experiments with
pixel values from others.

Cropping experiments to a common duration
-----------------------------------------

CROP_TO_SHORTEST_EXPERIMENT controls whether all time-series plots are limited
to the shortest experiment:

    CROP_TO_SHORTEST_EXPERIMENT = True

When True, the script:

1. checks the final frame available in the global and regional CSV files;
2. finds the shortest shared frame range;
3. crops every experiment to that range before plotting.

This prevents a longer acquisition from extending beyond the other experiments.
However, the script crops by frame number, not necessarily by identical elapsed
time. If acquisition schedules differ between experiments, verify that their
frame-to-time mappings are comparable before interpreting the plots.

Set this to False to retain the full duration of every experiment:

    CROP_TO_SHORTEST_EXPERIMENT = False

Plot and export settings
------------------------

SAVE_PDF
    When True, every plot is saved as both PNG and PDF.

SAVE_LEGENDS_SEPARATELY
    When True, plot legends are removed from the main figure and saved as
    separate files. This is useful when assembling multi-panel thesis figures.

DPI
    PNG export resolution. The current setting is 1200 dpi.

FONT_SIZE
    All figure text is set to 16-point bold.

FIGSIZE_SINGLE and FIGSIZE_WIDE
    Physical figure sizes in inches.

LINEWIDTH and MARKER_SIZE
    Control the appearance of plotted lines and markers.

The figures are saved directly and are not displayed interactively.

Files read by the script
------------------------

1. global_metrics_relative_to_prebleach.csv

   Required columns used in this comparison include:

       frame
       time_s
       phase
       solidity
       area_change_vs_prebleach
       perimeter_change_vs_prebleach

2. regional_metrics_relative_to_prebleach.csv

   Required columns used in this comparison include:

       frame
       time_s
       phase
       bleached_mean_abs_radial_displacement_um or _px
       nonbleached_mean_abs_radial_displacement_um or _px
       bleached_mean_abs_local_strain
       nonbleached_mean_abs_local_strain

The script also contains fallback column names for older output versions where
possible.

Meaning of the four plots
-------------------------

1. Global contour solidity

   Solidity is the contour area divided by the area of its convex hull. Values
   near 1 indicate a nearly convex shape. Lower values indicate stronger
   concavities or irregular deformation.

2. Area and perimeter change relative to prebleach

   The ablation-analysis output stores fractional changes relative to the mean
   prebleach value. The comparison script multiplies these values by 100 and
   plots them as percentages.

   Zero therefore represents the mean prebleach reference.

3. Mean absolute radial displacement

   This reports the magnitude of motion along radial directions defined from
   the mean prebleach contour. Bleached and non-bleached regions are plotted
   separately.

   Because the absolute value is used, inward and outward displacement both
   contribute positively.

4. Mean absolute local strain

   This measures the magnitude of local changes in contour-segment length
   relative to the mean prebleach segment lengths. The fractional strain values
   are multiplied by 100 and plotted as percentages.

   Bleached and non-bleached regions are shown separately.

First-postbleach marker
-----------------------

When the input CSV files contain both "phase" and "time_s", the script places a
vertical dashed line at the earliest first-postbleach time found across the
experiments.

Before accepting this marker, confirm that all experiments use compatible time
axes and that the bleach event occurs at the same logical point in each series.

Output files
------------

The main plot files are:

    comparison_global_contour_solidity.png
    comparison_area_perimeter_change_relative_to_prebleach.png
    comparison_mean_absolute_radial_displacement.png
    comparison_mean_absolute_local_strain.png

When SAVE_PDF is True, corresponding PDF files are also generated.

When SAVE_LEGENDS_SEPARATELY is True, separate legend files are created with
"_legend" appended to the filename.

How to run
----------

1. Confirm that ablation_analysis.py has been run successfully for every GUV.
2. Inspect the quality-control outputs from every individual analysis.
3. Open mechanical_perturbation_compare_samples.py.
4. Update every experiment name and folder path.
5. Update OUTPUT_FOLDER.
6. Confirm PREFERRED_LENGTH_UNIT.
7. Confirm whether cropping to the shortest experiment is appropriate.
8. Run:

       python mechanical_perturbation_compare_samples.py

9. Inspect the terminal for messages describing loaded files, cropping, and
   saved outputs.
10. Inspect the output figures and separate legends.

Quality-control requirements before comparison
----------------------------------------------

The comparison script assumes that the individual CSV values are valid. It does
not recheck contour tracking or bleached-region detection.

Before comparing experiments, verify for every ablation-analysis run:

1. the TIFF stack was correctly registered;
2. the JFilament contours were valid;
3. the bleached region was detected correctly;
4. contour mapping was stable;
5. the same contour-processing settings were used;
6. the same definition of prebleach frames was used;
7. physical pixel calibration was correct;
8. time-axis settings were correct;
9. any manual adjustments were documented;
10. the relevant output CSV files were not manually altered.

Comparability considerations
----------------------------

Use caution when comparing experiments that differ in:

    number of prebleach frames
    frame interval
    total acquisition duration
    pixel size
    microscope settings
    contour-point number
    bleached-region detection mode
    smoothing parameters
    manual versus automatic arc assignment

For the most defensible comparison, keep the settings in ablation_analysis.py
identical except where acquisition metadata requires a change.

Common errors
-------------

Folder not found
    Check the path in the EXPERIMENTS list. It must point to the
    guv_deformation_analysis folder, not to the TIFF itself.

Global metrics file not found
    Confirm that global_metrics_relative_to_prebleach.csv exists in the
    experiment folder.

Regional metrics file not found
    Confirm that regional_metrics_relative_to_prebleach.csv exists in the
    experiment folder.

Expected column not found
    The CSV may have been generated by an incompatible code version. Check the
    available columns printed in the error message and rerun the current
    ablation_analysis.py if necessary.

Mixed displacement units
    Confirm that all experiments contain the requested _um or _px columns and
    that PREFERRED_LENGTH_UNIT is set correctly.

Plot ends earlier than expected
    CROP_TO_SHORTEST_EXPERIMENT is probably True. Set it to False to retain the
    full duration of every experiment.

Legends are missing from the main plot
    This is expected when SAVE_LEGENDS_SEPARATELY is True. Open the associated
    _legend image or PDF.

Recommended directory organization
----------------------------------

A clear project structure is:

    project/
        experiment_1/
            registered_stack.tif
            contours.snakes
            guv_deformation_analysis/
                global_metrics_relative_to_prebleach.csv

                pointwise_metrics_relative_to_prebleach.csv
                quality-control outputs

        experiment_2/
            registered_stack.tif
            contours.snakes
            guv_deformation_analysis/
                ...

        comparison/
            comparison_global_contour_solidity.png
            comparison_area_perimeter_change_relative_to_prebleach.png
            comparison_mean_absolute_radial_displacement.png
            comparison_mean_absolute_local_strain.png

Reproducibility recommendations
-------------------------------

Archive together:

1. the exact ablation_analysis.py version used for every experiment;
2. mechanical_perturbation_compare_samples.py;
3. both README files;
4. all original and registered TIFF files;
5. all contour files;
6. every individual analysis output folder;
7. the final comparison figures;
8. a record of analysis settings and any manual changes.

Summary
-------

ablation_analysis.py performs the full image- and contour-level analysis for one
GUV. mechanical_perturbation_compare_samples.py then reads the resulting global
and regional CSV files from several GUVs and produces standardized comparison
plots. The comparison is only as reliable as the quality control and parameter
consistency of the individual ablation analyses.
